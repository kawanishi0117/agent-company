# Waiver（例外承認）ポリシー

## 概要

Waiverは品質基準の一時的な例外を認める仕組み。乱用を防ぐため、厳格なルールを設ける。

## Waiver発行条件

以下をすべて満たす場合のみ発行可能：

1. **緊急性**: ビジネス上の緊急性が高い
2. **一時的**: 恒久的な例外ではない
3. **代替策**: リスク軽減策が明記されている
4. **期限**: 解消期限が設定されている
5. **フォロー**: 解消タスクが作成されている

## Waiverテンプレート

```markdown
# Waiver: [タイトル]

## 申請日

YYYY-MM-DD

## 申請者

[エージェント名]

## 対象

[例外を求める品質基準]

## 理由

[なぜ例外が必要か]

## 緊急性

[なぜ今すぐ必要か]

## 代替策

[リスク軽減のために何をするか]

## 期限

YYYY-MM-DD

## フォロータスク

- [ ] [解消のためのタスク1]
- [ ] [解消のためのタスク2]

## 承認者

[Quality Authority]

## ステータス

[ ] 申請中 / [ ] 承認 / [ ] 却下 / [ ] 解消済み
```

## Waiver管理

### 保存場所

`workflows/waivers/YYYY-MM-DD-[title].md`

### レビュー頻度

- 週次: 未解消Waiverの確認
- 月次: Waiver傾向分析

### 期限超過時

1. Quality Authorityに自動通知
2. 延長申請または即時解消を要求
3. 3回延長で強制エスカレーション

## 禁止事項

- 期限なしのWaiver
- フォロータスクなしのWaiver
- 同一理由での繰り返しWaiver
- セキュリティ関連の長期Waiver
