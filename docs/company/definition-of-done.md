# Definition of Done (DoD)

AgentCompanyにおける成果物の完了基準を定義する。

## 成果物レポートの必須セクション

全ての成果物レポートには以下のセクションが必須：

### 1. 目的
- 変更の目的を明確に記述
- なぜこの変更が必要なのかを説明

### 2. 変更点
- 具体的な変更内容をリスト化
- 影響を受けるファイル・コンポーネントを明記

### 3. テスト結果
- ユニットテストの実行結果
- テストカバレッジ（可能な場合）
- 失敗したテストがある場合はその理由

### 4. E2E結果
- E2Eテストの実行結果
- スクリーンショット/動画へのリンク（該当する場合）
- 手動検証の結果（該当する場合）

### 5. ロールバック
- 問題発生時の復旧手順
- 影響を最小化するための手順

### 6. リスク / 未検証
- 残存するリスク
- 未検証の項目
- 今後の対応予定

## 品質判定基準

### PASS（合格）
- 全ての必須セクションが記載されている
- テストが全て通過している
- リスクが適切に文書化されている
- ロールバック手順が明確

### FAIL（不合格）
- 必須セクションが欠けている
- テストが失敗している
- 重大なリスクが未対処
- ロールバック手順が不明確

### WAIVER（例外承認）
- 正当な理由がある
- 期限が設定されている
- フォロータスクが明確
- Quality Authorityの承認がある

## 成果物レポートテンプレート

```markdown
# 成果物レポート: [タイトル]

## 目的
[変更の目的を記述]

## 変更点
- [変更1]
- [変更2]

## テスト結果
- 実行コマンド: `make test`
- 結果: PASS / FAIL
- カバレッジ: XX%

## E2E結果
- 実行コマンド: `make e2e`
- 結果: PASS / FAIL
- 成果物: [リンク]

## ロールバック
1. [手順1]
2. [手順2]

## リスク / 未検証
- [リスク1]
- [未検証項目1]
```
