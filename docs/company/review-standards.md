# レビュー基準

## Quality Authority 判定基準

### PASS（合格）

以下をすべて満たす場合：
- [ ] `make lint` 成功
- [ ] `make test` 成功（カバレッジ80%以上）
- [ ] `make e2e` 成功
- [ ] 成果物フォーマット準拠
- [ ] セキュリティ懸念なし

### FAIL（不合格）

以下のいずれかに該当する場合：
- 品質ゲート失敗
- 成果物フォーマット不備
- セキュリティリスクあり
- テストカバレッジ不足

### WAIVER（例外承認）

以下の条件をすべて満たす場合のみ発行可能：
- 緊急性が高い
- 代替策が明記されている
- 期限が設定されている
- フォロータスクが作成されている

## コードレビュー観点

### 必須チェック項目

1. **機能**: 要件を満たしているか
2. **テスト**: 適切なテストがあるか
3. **セキュリティ**: 脆弱性がないか
4. **可読性**: コメント・命名が適切か
5. **パフォーマンス**: 明らかな問題がないか

### 禁止事項

- ハードコードされた認証情報
- allowlist外パッケージの使用
- テストのスキップ（理由なし）
- 未処理のエラー

## 成果物レビュー

### 必須セクション確認

```markdown
## 目的
（変更の目的が明確か）

## 変更点
（何を変更したか具体的か）

## テスト結果
（make test の結果）

## E2E結果
（make e2e の結果）

## ロールバック
（問題時の復旧手順）

## リスク
（潜在的なリスクと対策）
```
